Project 1

ELT Pipeline

Project Overview:
1. Introduction & SQL Code Repository
2. Extract: Data Acquisition
3. Load: Database Initialization
4. Transform: Data Cleaning & Manipulation
5. Transform: Featured Engineering
6. Final Dataset

1. Introduction & SQL Code Repository
The objective of the ELT (Extract, Load and Transform) project was to prepare and transform raw data into a structured and analyzable format. As part of the project, CSV files were imported into a MySQL database, where transformation operations were performed, including data cleaning, normalization and feature engineering.
The final SQL code is available on my GitHub profile: LINK.
Below is a step-by-step text description of all actions carried out in this ELT project.

2. Extract: Data Acquisition
The project began with the acquisition of three distinct CSV datasets:
2.1 "World Population 1960-2023 by Country" Dataset: The first dataset detailed population statistics for 255 countries and regions by 1960 - 2023 period. 3. Load: Database Initialization
To establish a structured data processing environment, the energy_analysis database was created in MySQL, serving as the main repository for raw datasets.
Within the energy_analysis database, three tables were created — world_temperature, electricity_generation, and population_table — each mirroring the structure of the corresponding CSV sources to maintain the integrity of the original data.

4. Transform: Data Cleaning & Manipulation
The transformation phase began with aligning all datasets to an unified timeframe (2000–2022) to enable consistent cross-dataset analysis, requiring the removal of records outside this range. Next, structural inconsistencies were addressed by standardizing column names, data types, and table schemas across the three sources. Redundant or non-essential columns were eliminated to reduce complexity and focus on core analytical variables. A critical challenge was resolving country name discrepancies—for example, harmonizing variations like "United States" and "USA" —using a standardized country-code reference to ensure uniformity. Typographical errors and regional naming differences were systematically corrected to avoid fragmentation in geographical analysis. To guarantee data integrity, duplicate entries were purged, and only countries present across all three datasets were retained, resulting in a refined subset of 160 countries to prevent incomplete or biased insights.
Finally, the cleaned datasets were merged into a single table, final_analysis_table, which consolidated yearly records of temperature, population, and energy generation into a cohesive structure optimized for analytical queries.

5. Transform: Featured Engineering
To enable robust potential analysis, three key features were derived:
5.1 Year-over-Year Changes: Absolute differences in population, temperature, and energy production compared to the prior year. As the result, the following columns have been added to the final table: population_growth, electro_growth and temp_growth
5.2 Percentage Year-over-Year Changes: Standardized growth/decline rates for each metric. As the result, the following columns have been added to the final table: population_growth_pct, temp_growth_pct and electro_growth_pct
5.3 Energy Production per Capita: Annual electricity production divided by population. As the result, the following column have been added to the final table: electro_per_capita_kwh

6. Final Dataset
As a final step, the SQL code includes basic data aggregation related to population, temperature, and energy generation data. This project focuses on data extraction and transformation, ensuring that the final dataset is structured and ready for export in CSV format, making it suitable for potential exploratory data analysis (EDA) and visualization of the results.
